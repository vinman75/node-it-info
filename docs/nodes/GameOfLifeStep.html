GameOfLifeStep

Category: Simualtion

Description: Calculates the next state of a grid based on 'Birth' and 'Survival' rules. Driven by 'Frame In' and can be reset via 'Reset In'. INITIALIZATION: - If 'Initial Grid In' is connected: Uses this grid upon reset or first step. - If 'Initial Grid In' is DISCONNECTED: Upon reset or first step, it generates its own grid using *fixed internal constants* (50x50, Seed 0, Density ~20%). These fixed constants are NOT user-configurable on this node; use 'GameOfLifeInitGridNode' for parametric initial grids. RULES: - Rules (e.g., 'B3/S23' for Conway's Life) are set as string attributes ('Birth Rules', 'Survival Rules') in the Node Editor (e.g., '3' and '23'). - These can be overridden by connecting lists of integers to 'Birth Rules In' or 'Survival Rules In' ports. BOUNDARY: - 'Boundary Mode' dropdown controls edge behavior (Wrap, Fixed Dead, Fixed Alive). Output is 'Current Grid Out' (ValueGrid).

Inputs:

  • Initial Grid In: ValueGrid
  • Frame In: int
  • Reset In: bool
  • Birth Rules In: list
  • Survival Rules In: list

Outputs:

  • Output: ValueGrid
